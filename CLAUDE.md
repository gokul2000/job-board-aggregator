# Claude Code Rules

## Git Workflow

- NEVER push directly to the `main` branch. All changes to `main` must go through a pull request.
- Always work on the `develop` branch or create feature branches from `develop`.
- When changes are ready for `main`, create a PR from `develop` (or feature branch) → `main`.
- Assign `gokul2000` as the reviewer on every PR.
- Do NOT merge PRs — wait for the owner to approve and merge.
- Do NOT force push to any branch unless explicitly asked.
